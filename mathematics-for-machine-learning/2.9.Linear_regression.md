# Linear Regression

*Source: Mathematics for Machine Learning, Deisenroth, Faisal, Ong (2020), Chapter 9*

---

## 9.1 Problem Formulation

Model: $y = \phi(\boldsymbol{x})^T\boldsymbol{\theta} + \epsilon$, $\epsilon \sim \mathcal{N}(0, \sigma^2)$, i.e.,
$$p(y \mid \boldsymbol{x}, \boldsymbol{\theta}) = \mathcal{N}(y \mid \phi(\boldsymbol{x})^T\boldsymbol{\theta}, \sigma^2)$$

"Linear regression" means linear in $\boldsymbol{\theta}$, not in $\boldsymbol{x}$. Feature map $\phi: \mathbb{R}^D \to \mathbb{R}^K$ can be nonlinear (polynomial, RBF, Fourier, etc.). The design matrix $\boldsymbol{\Phi} = [\phi(\boldsymbol{x}_1), \ldots, \phi(\boldsymbol{x}_N)]^T \in \mathbb{R}^{N \times K}$.

> **Intuition.** "Linear in the parameters" is the key constraint — it keeps the objective convex (a quadratic in $\boldsymbol{\theta}$), admitting a closed-form solution. The feature map $\phi$ is what gives the model expressive power: with $\phi(\boldsymbol{x}) = [1, x, x^2, \ldots, x^{M-1}]^T$ you get polynomial regression; with $\phi(\boldsymbol{x}) = [\sin(\omega_1 x), \cos(\omega_1 x), \ldots]$ you get Fourier regression. The model is still "linear regression" and inherits all the same analytical machinery.

> **Usage.** Linear regression (in this general sense) covers ridge regression, LASSO, polynomial regression, and any model where predictions are an inner product of fixed features and learned weights. The design matrix $\boldsymbol{\Phi}$ is the central object: its rank determines identifiability, its condition number determines numerical stability, and its column space determines what functions can be represented.

> **Practicality.** Feature engineering — choosing $\phi$ — is often more important than solver choice. RBF features (random Fourier features, Nyström approximation) can approximate kernel regression in $O(NK)$ instead of $O(N^2)$. Normalization of features matters: without it, features with different scales make the condition number of $\boldsymbol{\Phi}^T\boldsymbol{\Phi}$ large, causing numerical instability in the normal equations.

## 9.2 Parameter Estimation

### Maximum Likelihood Estimation (MLE)

Under i.i.d. Gaussian noise, NLL $\propto \|\boldsymbol{y} - \boldsymbol{\Phi}\boldsymbol{\theta}\|^2$. Setting gradient to zero:
$$\boldsymbol{\theta}_\text{ML} = (\boldsymbol{\Phi}^T\boldsymbol{\Phi})^{-1}\boldsymbol{\Phi}^T\boldsymbol{y}$$

Requires $\text{rk}(\boldsymbol{\Phi}) = K$ (full column rank). The Hessian $\boldsymbol{\Phi}^T\boldsymbol{\Phi} \succ 0$ confirms this is a global minimum.

**Noise variance estimate**: $\sigma^2_\text{ML} = \frac{1}{N}\|\boldsymbol{y} - \boldsymbol{\Phi}\boldsymbol{\theta}_\text{ML}\|^2$.

> **Intuition.** The normal equations $\boldsymbol{\Phi}^T\boldsymbol{\Phi}\boldsymbol{\theta} = \boldsymbol{\Phi}^T\boldsymbol{y}$ have a clean geometric reading (§9.4): they express that the residual $\boldsymbol{y} - \boldsymbol{\Phi}\boldsymbol{\theta}_\text{ML}$ is orthogonal to every column of $\boldsymbol{\Phi}$, i.e., the prediction is the orthogonal projection of $\boldsymbol{y}$ onto the column space of $\boldsymbol{\Phi}$. MLE under Gaussian noise is geometrically identical to finding the nearest point in the column space.

> **Usage.** Direct solution via the normal equations is the standard approach for small to medium $K$ (< 10,000 features). For large $K$ or ill-conditioned $\boldsymbol{\Phi}^T\boldsymbol{\Phi}$, use the pseudoinverse via SVD: $\boldsymbol{\theta}_\text{ML} = \boldsymbol{V}\boldsymbol{\Sigma}^+\boldsymbol{U}^T\boldsymbol{y}$, which handles rank-deficiency gracefully by setting near-zero singular values to zero.

> **Practicality.** Never invert $\boldsymbol{\Phi}^T\boldsymbol{\Phi}$ directly — use `np.linalg.lstsq` or `scipy.linalg.solve` with the normal equations. For $N \gg K$: form and solve the $K \times K$ normal equations ($O(NK^2 + K^3)$). For $K \gg N$: use the dual form or SVD of $\boldsymbol{\Phi}$ directly ($O(N^2 K)$). The MLE noise variance $\sigma^2_\text{ML}$ is biased; the unbiased estimate divides by $N - K$ (degrees of freedom correction).

### Overfitting

Polynomial degree $M$: low $M$ underfits; $M \geq N$ makes $\boldsymbol{\Phi}^T\boldsymbol{\Phi}$ non-invertible (infinitely many solutions). Optimal $M$ found via test RMSE or cross-validation. Model complexity should scale with dataset size.

> **Intuition.** When $K \geq N$ there are more free parameters than data points: the model can interpolate the training data exactly (zero training error) while fitting the noise. The resulting function oscillates wildly between data points — high variance, low bias. This is the classical bias-variance tradeoff: increasing $M$ decreases bias but increases variance; the optimal $M$ minimizes total expected error on new data.

> **Practicality.** The bias-variance tradeoff is visible on a validation curve: training error decreases monotonically with $M$; test error has a U-shape. In modern deep learning, models are routinely over-parameterized ($K \gg N$), yet they generalize well due to implicit regularization from SGD and architecture inductive biases — this is the "double descent" phenomenon that breaks the classical U-curve intuition at very large $K$.

### MAP Estimation

Prior $p(\boldsymbol{\theta}) = \mathcal{N}(\mathbf{0}, b^2\boldsymbol{I})$:
$$\boldsymbol{\theta}_\text{MAP} = (\boldsymbol{\Phi}^T\boldsymbol{\Phi} + \tfrac{\sigma^2}{b^2}\boldsymbol{I})^{-1}\boldsymbol{\Phi}^T\boldsymbol{y}$$

Always invertible regardless of rank. Equivalent to **ridge regression** with $\lambda = \sigma^2/b^2$. Lasso ($L_1$ prior) promotes sparsity: $\boldsymbol{\theta}_\text{LASSO} = \arg\min \|\boldsymbol{y} - \boldsymbol{\Phi}\boldsymbol{\theta}\|^2 + \lambda\|\boldsymbol{\theta}\|_1$.

> **Intuition.** Adding $\lambda\boldsymbol{I}$ to $\boldsymbol{\Phi}^T\boldsymbol{\Phi}$ shifts all eigenvalues up by $\lambda$, making the matrix invertible regardless of rank and numerically stable regardless of conditioning. Ridge shrinks all coefficients toward zero proportionally; LASSO shrinks smaller coefficients to exactly zero (sparsity) because the $L_1$ ball's corners sit on coordinate axes. The geometric picture: ridge keeps the solution within an $L_2$ ball (smooth shrinkage), LASSO keeps it within an $L_1$ ball (induces corners at sparse solutions).

> **Usage.** Ridge is the go-to when all features are expected to contribute and you want a stable solution. LASSO is preferred for feature selection in high-dimensional settings ($K \gg N$): the $L_1$ penalty zeros out irrelevant features, producing a sparse $\boldsymbol{\theta}$ that is interpretable. Elastic net ($\alpha\|\boldsymbol{\theta}\|_1 + (1-\alpha)\|\boldsymbol{\theta}\|_2^2$) combines both — useful when features are correlated (ridge handles groups, LASSO picks one from each group arbitrarily).

> **Practicality.** Ridge has a closed form; LASSO requires iterative solvers. The canonical LASSO solver is coordinate descent (cycle through coordinates, update each analytically using the soft-thresholding operator). `sklearn.linear_model.Ridge`, `Lasso`, `ElasticNet` are standard. The regularization path (solution as a function of $\lambda$) can be computed efficiently with LARS for LASSO. Always tune $\lambda$ via cross-validation; the optimal $\lambda$ scales roughly as $\sigma^2/b^2$, so understanding the prior variance $b^2$ can initialize the search range.

## 9.3 Bayesian Linear Regression

Places a full distribution over $\boldsymbol{\theta}$ rather than a point estimate, naturally handling uncertainty and avoiding overfitting.

### Model
- Prior: $p(\boldsymbol{\theta}) = \mathcal{N}(\boldsymbol{m}_0, \boldsymbol{S}_0)$
- Likelihood: $p(\boldsymbol{y} \mid \boldsymbol{\Phi}, \boldsymbol{\theta}) = \mathcal{N}(\boldsymbol{\Phi}\boldsymbol{\theta}, \sigma^2\boldsymbol{I})$

### Posterior (closed form via conjugacy)

$$p(\boldsymbol{\theta} \mid \mathcal{X}, \mathcal{Y}) = \mathcal{N}(\boldsymbol{m}_N, \boldsymbol{S}_N)$$
$$\boldsymbol{S}_N = (\boldsymbol{S}_0^{-1} + \sigma^{-2}\boldsymbol{\Phi}^T\boldsymbol{\Phi})^{-1}$$
$$\boldsymbol{m}_N = \boldsymbol{S}_N(\boldsymbol{S}_0^{-1}\boldsymbol{m}_0 + \sigma^{-2}\boldsymbol{\Phi}^T\boldsymbol{y})$$

Derived by completing the square in the joint Gaussian. The posterior precision ($\boldsymbol{S}_N^{-1}$) is the sum of prior precision and data precision — information additivity.

With $\boldsymbol{m}_0 = \mathbf{0}$, $\boldsymbol{S}_0 = b^2\boldsymbol{I}$: $\boldsymbol{m}_N$ reduces to $\boldsymbol{\theta}_\text{MAP}$.

> **Intuition.** The posterior mean $\boldsymbol{m}_N$ is a precision-weighted interpolation between the prior mean $\boldsymbol{m}_0$ and the data-driven estimate. With no data ($N = 0$): $\boldsymbol{S}_N = \boldsymbol{S}_0$, $\boldsymbol{m}_N = \boldsymbol{m}_0$. With infinite data: the posterior concentrates on the MLE. The posterior precision is additive in information: each data point contributes $\sigma^{-2}\phi(\boldsymbol{x}_n)\phi(\boldsymbol{x}_n)^T$ to the total precision, shrinking posterior uncertainty.

> **Usage.** Bayesian LR is the analytic foundation for Gaussian processes (§9.5). It also provides the exact sequential Bayesian update rule: if you have a current posterior $\mathcal{N}(\boldsymbol{m}, \boldsymbol{S})$ and observe a new data point, the update equations give the new posterior in closed form — this is online Bayesian learning. This is the basis for Kalman filtering (Bayesian linear regression for dynamical systems).

> **Practicality.** The $K \times K$ matrix inversion for $\boldsymbol{S}_N$ costs $O(K^3)$. For large $K$ (many features), use the matrix inversion lemma (Woodbury identity) to work with the $N \times N$ dual matrix instead: $\boldsymbol{S}_N = \boldsymbol{S}_0 - \boldsymbol{S}_0\boldsymbol{\Phi}^T(\boldsymbol{\Phi}\boldsymbol{S}_0\boldsymbol{\Phi}^T + \sigma^2\boldsymbol{I})^{-1}\boldsymbol{\Phi}\boldsymbol{S}_0$. This is $O(N^3)$ rather than $O(K^3)$ — the same switch as from primal to dual in SVMs.

### Posterior Predictive Distribution

$$p(y_* \mid \boldsymbol{x}_*, \mathcal{X}, \mathcal{Y}) = \mathcal{N}(\phi(\boldsymbol{x}_*)^T\boldsymbol{m}_N,\; \phi(\boldsymbol{x}_*)^T\boldsymbol{S}_N\phi(\boldsymbol{x}_*) + \sigma^2)$$

The predictive variance decomposes into **model uncertainty** $\phi^T\boldsymbol{S}_N\phi$ (epistemic, shrinks with data) and **observation noise** $\sigma^2$ (aleatoric, irreducible). Predictive variance is larger far from training data — this is the correct inductive bias.

> **Intuition.** Epistemic uncertainty ($\phi^T\boldsymbol{S}_N\phi$) is uncertainty about the model — it shrinks as more data constrains the posterior. Aleatoric uncertainty ($\sigma^2$) is uncertainty about the data-generating process itself — it is irreducible and does not shrink with more data. This decomposition is exactly what you want for decision-making under uncertainty: you can reduce epistemic uncertainty by collecting more data in uncertain regions, but aleatoric uncertainty is fundamental.

> **Usage.** The predictive variance as a function of $\boldsymbol{x}_*$ is the basis for **active learning**: query the point $\boldsymbol{x}_*$ with the highest predictive variance to maximally reduce model uncertainty. It is also the basis for Bayesian optimization: the acquisition function (UCB, EI) uses the predictive mean and variance to balance exploration and exploitation.

> **Practicality.** Computing predictive variance for a single test point costs $O(K^2)$ (matrix-vector product with $\boldsymbol{S}_N$). For a batch of $M$ test points, computing the full predictive covariance matrix costs $O(MK^2 + M^2K)$. In the Gaussian process limit (infinite $K$), this becomes $O(N^2)$ per test point, which is the standard GP inference cost.

### Marginal Likelihood

$$p(\mathcal{Y} \mid \mathcal{X}) = \mathcal{N}(\boldsymbol{y} \mid \boldsymbol{\Phi}\boldsymbol{m}_0,\; \boldsymbol{\Phi}\boldsymbol{S}_0\boldsymbol{\Phi}^T + \sigma^2\boldsymbol{I})$$

Used for Bayesian model selection (choosing feature map, degree, kernel). Maximizing this over hyperparameters is called **evidence maximization** (empirical Bayes / type-II MLE).

> **Intuition.** The marginal likelihood is the probability of the observed labels under the model, with parameters integrated out. It balances fit (models that predict the data well are rewarded) against complexity (models that spread probability over many possible datasets are penalized). The covariance structure $\boldsymbol{\Phi}\boldsymbol{S}_0\boldsymbol{\Phi}^T + \sigma^2\boldsymbol{I}$ encodes prior correlations between outputs: $[\boldsymbol{\Phi}\boldsymbol{S}_0\boldsymbol{\Phi}^T]_{ij} = \phi(\boldsymbol{x}_i)^T\boldsymbol{S}_0\phi(\boldsymbol{x}_j)$ is a positive definite kernel — this is the GP connection.

> **Usage.** Evidence maximization (maximize $\log p(\mathcal{Y} \mid \mathcal{X})$ over $\lambda = \sigma^2/b^2$) automatically selects the regularization strength without cross-validation. In Gaussian process regression, this is the standard method for learning kernel hyperparameters. It also provides a principled score for comparing models of different complexity (polynomial degree, feature type).

> **Practicality.** Computing $\log p(\mathcal{Y} \mid \mathcal{X})$ requires evaluating the log determinant of the $N \times N$ matrix $\boldsymbol{\Phi}\boldsymbol{S}_0\boldsymbol{\Phi}^T + \sigma^2\boldsymbol{I}$, which costs $O(N^3)$ via Cholesky. This is the bottleneck that limits exact GP inference to $N \lesssim 10^4$; sparse GP approximations (inducing points) reduce this to $O(NM^2)$ where $M \ll N$.

## 9.4 MLE as Orthogonal Projection

$\boldsymbol{\Phi}\boldsymbol{\theta}_\text{ML} = \boldsymbol{\Phi}(\boldsymbol{\Phi}^T\boldsymbol{\Phi})^{-1}\boldsymbol{\Phi}^T\boldsymbol{y}$ is the orthogonal projection of $\boldsymbol{y}$ onto the column space of $\boldsymbol{\Phi}$. The projection matrix $\boldsymbol{P} = \boldsymbol{\Phi}(\boldsymbol{\Phi}^T\boldsymbol{\Phi})^{-1}\boldsymbol{\Phi}^T$ satisfies $\boldsymbol{P}^2 = \boldsymbol{P}$. Residuals $\boldsymbol{y} - \boldsymbol{\Phi}\boldsymbol{\theta}_\text{ML}$ are orthogonal to the column space of $\boldsymbol{\Phi}$ — this is the geometric content of the normal equations.

> **Intuition.** Linear regression literally projects $\boldsymbol{y}$ onto the subspace of functions representable by $\boldsymbol{\Phi}$ (the column space). The residual vector is the component of $\boldsymbol{y}$ that lies outside that subspace — it is orthogonal to all representable functions by construction. The hat matrix $\boldsymbol{H} = \boldsymbol{P}$ maps $\boldsymbol{y}$ to fitted values $\hat{\boldsymbol{y}}$; its diagonal entries $H_{nn}$ are leverage scores — how much each data point influences its own prediction (high leverage = potential outlier).

> **Usage.** Leverage scores ($H_{nn}$) are used in influence analysis and robust regression: high-leverage points can dominate the projection and should be scrutinized. Cook's distance combines leverage and residuals to identify influential observations. The trace $\text{tr}(\boldsymbol{H}) = K$ equals the number of free parameters, which is why the unbiased variance estimate divides residuals by $N - K$.

> **Practicality.** The geometric view is indispensable for understanding what regression is doing: it is the minimum-norm residual in $L_2$. Any regularization method (ridge, LASSO, PCA regression) modifies this projection — ridge shrinks the projection in directions of low variance (small singular values of $\boldsymbol{\Phi}$); PCA regression truncates those directions entirely.

## 9.5 Further Reading

- **Generalized Linear Models (GLMs)**: non-Gaussian likelihoods via $p(y \mid \boldsymbol{x}) = \text{ExpFamily}(\eta = f(\boldsymbol{x}^T\boldsymbol{w}))$; logistic regression (Bernoulli + sigmoid), Poisson regression (log link). Foundational to neural networks.
- **Gaussian Processes**: nonparametric Bayesian regression — place a prior directly over functions via a kernel $k(\boldsymbol{x}, \boldsymbol{x}') = \phi(\boldsymbol{x})^T\boldsymbol{S}_0\phi(\boldsymbol{x}')$. Bayesian linear regression with infinite features (kernel trick).
- **LASSO / Sparse regression**: $L_1$ regularization for feature selection in high-dimensional settings; coordinate descent is the canonical solver.

> **Intuition.** GLMs generalize linear regression by replacing the Gaussian noise model with any exponential family distribution and linking the natural parameter to the linear predictor via a link function. This covers virtually all classical statistical models (logistic, Poisson, gamma regression) and is the mathematical foundation of the last layer of a neural network (softmax + cross-entropy = categorical GLM). The exponential family structure guarantees that the MLE gradient has the form $\boldsymbol{\Phi}^T(\boldsymbol{y} - \hat{\boldsymbol{y}})$ — residuals projected back through the features — regardless of the specific distribution.

> **Practicality.** GPs are the continuous analogue of Bayesian linear regression in an infinite-dimensional feature space. The kernel specifies the prior over functions: RBF kernel = smooth functions; Matérn-1/2 = rough; periodic kernel = periodic. GP hyperparameter tuning via evidence maximization is standard (GPyTorch, GPflow). For large $N$, sparse GP approximations (SVGP, KISS-GP) are necessary.
